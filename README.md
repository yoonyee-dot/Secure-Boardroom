# Secure-Boardroom
# 📌 프로젝트 제목
Secure-Boardroom

보안 텍스트 회의 터미널 및 무결성 백업 시스템
## 📖 프로젝트 소개 
Secure Boardroom은 기업 및 금융권의 투명한 의사결정과 회의 기록 무결성 보장을 위한 보안 텍스트 회의 터미널 및 자동 백업 시스템입니다.

단순한 채팅을 넘어 보안, 안건 의결, 기록 보존 기능을 통합 제공하며, Linux 환경의 시스템 프로그래밍 핵심 기술을 실무 수준으로 구현하는 것을 목표로 합니다.
## ✨ 주요 기능 
- **기능 1:** 다중 접속 보안 채팅 및 안건 투표 시스템
- **기능 2:** 로그 파일 암호화 및 접근 권한 제어
- **기능 3:** 회의록 자동 압축 및 무결성 백업

## 🚀 빌드 및 실행 방법 
본 프로젝트는 다중 접속을 위한 서버-클라이언트 구조로 동작하며, WSL 환경에서 `ngrok`을 활용한 외부 접속을 지원합니다. 테스트를 위해 여러 개의 터미널 창이 필요합니다.

### 1. 프로젝트 빌드
터미널을 열고 프로젝트 디렉토리로 이동한 뒤 코드를 컴파일합니다.
```bash
wsl
cd ~/secure_boardroom
make
```

### 2.서버 실행 및 포트 포워딩

#### 터미널1(서버 실행):
```bash
# 빌드된 서버 프로그램을 실행합니다. (기본 포트: 8080)
wsl
cd ~/secure_boardroom
./server
```
#### 터미널 2 (ngrok 실행):
새로운 터미널을 열고 아래 명령어를 입력하여 8080 포트를 포워딩합니다.
```bash
wsl
cd ~/secure_boardroom
ngrok tcp 8080
```

### 3. 클라이언트 접속
#### 터미널 3 (클라이언트 실행):
새로운 터미널을 열고, 앞서 확인한 ngrok 주소와 포트를 사용하여 클라이언트를 실행합니다.
```bash
wsl
cd ~/secure_boardroom
./client
```
